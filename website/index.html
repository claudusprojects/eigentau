Process

How Eigentau thinks

Every query passes through four stages. Each stage sharpens the result. Each result sharpens the system.

01

Decompose

Complex tasks broken into atomic sub-tasks by AI. Each sub-task mapped to a cognitive faculty.

02

Route

Each sub-task matched to optimal Bittensor subnet(s) using learned routing weights and live performance data.

03

Synthesize

Results from multiple subnets combined, conflicts resolved, coherent unified answer assembled.

04

Learn

Routing patterns scored against outcomes. Weights adjusted via EMA blending. The system improves with every query.

Architecture

Mixture of Subnets

A learned gating network routes each task to the optimal combination of Bittensor's specialized subnets.

Like a Mixture of Experts model, Eigentau's router learns which subnet combinations produce the best results for each task type. The gating network adjusts weights via exponential moving average blending — 80% current knowledge, 20% new signal — ensuring stable, continuous improvement without catastrophic forgetting.

Learning Engine

Self-Evolving Intelligence

Every query teaches the router. Eigentau tracks which subnet combinations produce the best results and continuously optimizes its routing weights.

Outcome Tracking

Every routed query is scored. The system measures which subnet combinations produce the highest quality results for each task type.

Signal Correlation

Pearson correlation between routing signals and outcomes. Identifies which subnet performance metrics actually predict result quality.

EMA Weight Blending

Routing weights adjust via exponential moving average — 80% current knowledge, 20% new data. Stable improvement without catastrophic shifts.

Regime Detection

Detects shifts in network state — subnet congestion, performance degradation, new subnet launches — and adapts routing accordingly.
